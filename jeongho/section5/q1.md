## Core Componet 장애 해결하기

### Solution

```
# 1. pod 상태 확인 (에러 날 것)
k get pods -A

# 2. kubelet log 조회 (에러나는 것 확인 가능)
sudo journalctl -u kubelet

# 문제 힌트 중 외부 etcd server를 쓰다가 바꿨다고 함
# api-server가 etcd랑 연결되어 있다는 사실을 알고 있어야 함

# 3. api-server yaml 확인
sudo vi /etc/kubernetes/manifests/kube-apiserver.yaml

# --etcd-servers=https://192.168.56.41:2379 <- 이 ip가 외부 ip로 되어있어서 문제 발생127.0.0.1 로컬호스트로 변경

# 4. 다시 pod 조회
k get pods -A

# 파드가 조회는 되지만 component 에러나는 것 확인 가능 특히 kube-scheduler 에러
kube-system       kube-scheduler-k8s-master                   0/1     UnexpectedAdmissionError

# 5. kube-scheduler.yaml 조회
 # It is recommended to allocate kube-scheduler's resources to 10% of the worker node's resources.
이런 내용 있음

# 6. node 스펙 확인
k describe nodes
Capacity:
  cpu:                4
  ephemeral-storage:  30259180Ki
  hugepages-2Mi:      0
  memory:             3986664Ki
  pods:               110
Allocatable:
  cpu:                4
  ephemeral-storage:  27886860242
  hugepages-2Mi:      0
  memory:             3884264Ki
  pods:               110

# node cpu 4코어 현재 kube-scheduler도 4코어임 이걸 10%로 줘야함
# 7. kube-scheduler.yaml 수정
 cpu: 4 -> 400m

# 8. 재기동
sudo systemctl restart kubelet

# 9. pod 재조회
k get pods -A
```
