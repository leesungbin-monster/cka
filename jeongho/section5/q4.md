## PriorityClass 생성하기

### Solution

```
# 1. priorityclass 조회
k get priorityclass

# 기본 priorityclass
NAME                      VALUE        GLOBAL-DEFAULT   AGE   PREEMPTIONPOLICY
system-cluster-critical   2000000000   false            36d   PreemptLowerPriority
system-node-critical      2000001000   false            36d   PreemptLowerPriority

# 2. pod 조회
k get pods -n priority

NAME                            READY   STATUS    RESTARTS   AGE
busybox-logger-668456cb-cqv4z   0/1     Pending   0          18s
busybox-logger-668456cb-qcsd5   0/1     Pending   0          18s
busybox-logger-668456cb-xpvs2   0/1     Pending   0          18s
others-748b8ff466-4psfk         1/1     Running   0          19s
others-748b8ff466-f7n8s         1/1     Running   0          19s
others-748b8ff466-fx7wh         1/1     Running   0          19s

# 3. priortyclass 생성 (공식문서 참고)
k apply -f - <<EOF
apiVersion: scheduling.k8s.io/v1
kind: PriorityClass
metadata:
  name: high-priority
value: 999999999
EOF

# 4. deployment yaml 수정
# container와 같은 레벨에    
priorityClassName: high-priority <- 추가

# 5. pod 재조회
k get pods -n priority
```
