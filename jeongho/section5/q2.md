## 로그 출력을 위한 Sidecar 생성하기

### Solution

```
# 1. pod 조회 (현재 컨테이너 하나)
k get pods
NAME                                  READY   STATUS    RESTARTS   AGE
synergy-deployment-6cdb65946b-rs8rc   1/1     Running   0          26s

# 2. deployment yaml 확인
k edit deploy

# siedcar 컨테이너 없으므로 여기에 추가

# 3. sidecar container 추가 (공식문서 참고)
         volumeMounts:
            - name: data
              mountPath: /var/log
        - name: sidecar
          image: busybox:stable
          restartPolicy: Always
          command: ['sh', '-c', 'tail -n+1 -f /var/log/synergy-deployment.log']
          volumeMounts:
            - name: data
              mountPath: /var/log
      volumes:
        - name: data
          emptyDir: {}

# 문제에 맞게 들여쓰기 수정
deployment.apps/synergy-deployment edited <- 뜨면 잘 수정한 것

# 4. pod 조회
synergy-deployment-64488f46b8-6ggsp   2/2     Running       0          30s
synergy-deployment-6cdb65946b-rs8rc   1/1     Terminating   0          5m37s
# 기존 pod 내리고 새 pod 올림. 새 pod는 container 2개

# 5. log 조회
k logs [pod 이름] -c [container 이름]
```
