## **CPU & Memory 재설정하기**

### Solution

```
# 1. pod 조회
k get pods -n relative-fawn

# pod 한개가 안뜨고 있음
mariadb-77b7cfc86c-64qtz     1/1     Running   0          14s
wordpress-6f4fc6c46f-2k982   1/1     Running   0          13s
wordpress-6f4fc6c46f-5vgfn   1/1     Running   0          13s
wordpress-6f4fc6c46f-gttrm   0/1     Pending   0          13s

# 2. node spec 조회
k get nodes

Allocatable:
  cpu:                4
  ephemeral-storage:  27886860242
  hugepages-2Mi:      0
  memory:             3884280Ki
  pods:               110
# 3. deployment yaml 조회
        resources:
          limits:
            cpu: "1"
            memory: 1000Mi
          requests:
            cpu: "1"
            memory: 1000Mi

# 현재 node에 할당된 cpu를 초과해서 pod가 안뜨는 것
# deployment는 새 pod를 생성하고 기존 pod를 삭제함
# 하지만 자원이 부족한 상태이므로 yaml 수정해도 새 pod 안뜸
# replicas 3 -> 0 -> 3 으로 수정하며 재배포해서 새 pod 잘 뜨게 하기

# yaml 수정 후 재배포
          requests:
            cpu: 300m
            memory: 300Mi
```
