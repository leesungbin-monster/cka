## NetworkPolicy 생성하기

### Solution

```
# 1. networkpolicy 확인
k get networkpolicy -n backend

# 2. networkpolicy 내용 확인
k edit networkpolicy deny-all -n backend

# 3. 각 ns pod 및 label 확인
k get pod -n frontend
k get pod -n frontend --show-labels

k get pod -n backend
k get pod -n backend --show-labels

# 4. curl로 ns 간 통신 확인
k exec -n frontend frontend-6d64f59cc5-6l8lq -it -- curl 20.108.82.243:8080

# 현재 Ingress, Egress 설정에 아무 값도 없어서 모든 ingress egress 막혀있음

#  5. ~/netpol 폴더 확인
[cka0001@k8s-master ~]$ ls ~/netpol
netpol1.yaml  netpol2.yaml  netpol3.yaml

# 6. 내용 확인 후 문제 조건에 맞는 조건의 yaml로 networkpolicy 생성
k apply -f netpol/netpol2.yaml
networkpolicy.networking.k8s.io/netpol2 created
```

### Result

```
[cka0001@k8s-master ~]$ k exec -n frontend frontend-6d64f59cc5-6l8lq -it -- curl 20.108.82.243:8080
hello
```

### 기타

만약 curl로 트래픽 안간다면 label 확인

```
# namespace label 확인
k get ns frontend --show-labels
NAME       STATUS   AGE   LABELS
frontend   Active   54m   kubernetes.io/metadata.name=frontend,name=frontend

# pod label 확인
k get pod -n frontend frontend-6d64f59cc5-6l8lq  --show-labels
NAME                        READY   STATUS    RESTARTS   AGE   LABELS
frontend-6d64f59cc5-6l8lq   1/1     Running   0          55m   app=frontend,pod-template-hash=6d64f59cc5
```
